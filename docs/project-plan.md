# AI 驱动本体自动构建平台 — 项目计划

## 一、项目概述

构建一个以 **AI 驱动** 为核心的 **本体自动构建平台**，通过五层产品架构实现从数据感知到智能决策、再到业务执行与应用的完整闭环。

- **项目代号**：OntoMind（暂定）
- **核心理念**：让数据自动生长为本体，让本体自动驱动决策
- **目标用户**：数据工程师、风控/营销策略分析师、业务决策者

---

## 二、五层产品架构

```
┌─────────────────────────────────────────────────────────┐
│                     应用层 (Application)                  │
│          AIbi 智能分析产品 · 用户数据可视化产品            │
├─────────────────────────────────────────────────────────┤
│                     执行层 (Execution)                    │
│        策略下发引擎 → 风控系统 · 营销系统 · 其他业务系统    │
├─────────────────────────────────────────────────────────┤
│                     决策层 (Decision)                     │
│      特征挖掘 · ML模型训练 · 规则策略生成 · 在线执行       │
├─────────────────────────────────────────────────────────┤
│                     认知层 (Cognition)                    │
│           本体图谱构建 · 知识推理 · 语义理解               │
├─────────────────────────────────────────────────────────┤
│                     感知层 (Perception)                   │
│  数据源连接器 · 文档解析 · 数据仓库 · 数据资产 · 代码库    │
└─────────────────────────────────────────────────────────┘
```

### 2.1 感知层 (Perception Layer)
> **定位**：平台的信息入口，连接一切数据源

| 能力模块 | 描述 | 技术方向 |
|---------|------|---------|
| 数据源连接器 | 支持 JDBC/ODBC 连接关系型数据库（MySQL、PostgreSQL、Oracle 等） | Spring Boot + MyBatis |
| 文档解析引擎 | 上传并解析 PDF、Word、Excel、Markdown、TXT 等非结构化文档 | Apache Tika / Unstructured |
| 大数据连接器 | 对接数据仓库（Hive、ClickHouse）、数据湖（Iceberg/Hudi） | Spark / Trino |
| 数据资产系统 | 接入企业数据资产目录、元数据管理系统 | REST API 集成 |
| 代码库连接器 | 对接 Git 仓库（GitHub/GitLab），解析代码结构与注释 | Git API + AST 解析 |
| 实时数据流 | 对接 Kafka/Pulsar 消息队列，支持实时数据感知 | Kafka Connect |

**核心功能**：
- 统一数据源注册与连接管理
- 数据源元数据自动发现与采集
- 文档智能分块（Chunking）与向量化（Embedding）
- 数据血缘追踪

### 2.2 认知层 (Cognition Layer)
> **定位**：将感知层获取的信息提炼为结构化的本体知识图谱

| 能力模块 | 描述 | 技术方向 |
|---------|------|---------|
| 本体构建引擎 | 从结构化/非结构化数据中抽取实体、关系、属性 | LLM + NLP 管道 |
| 知识图谱存储 | 存储实体-关系-属性三元组 | Neo4j / NebulaGraph |
| 语义理解 | 基于本体进行语义搜索与推理 | GraphRAG |
| 本体融合 | 多源本体对齐、消歧、融合 | Ontology Alignment |
| 向量知识库 | 文档/代码的向量化存储与检索 | Milvus / Qdrant |

**核心功能**：
- 实体识别（NER）与关系抽取（RE）
- 自动本体生成与版本管理
- 图谱可视化探索
- 语义级数据检索

### 2.3 决策层 (Decision Layer)
> **定位**：基于本体图谱与业务目标，生成可执行的策略与规则

| 能力模块 | 描述 | 技术方向 |
|---------|------|---------|
| 特征挖掘 | 从本体图谱中自动发现有效特征 | Featuretools / 自定义 |
| ML 模型训练 | 风控模型、营销模型的训练与管理 | MLflow / Scikit-learn / XGBoost |
| 规则策略引擎 | 生成可在线执行的 if-then 规则 | Drools / 自研规则引擎 |
| 策略评估 | A/B 测试、效果评估、策略回测 | 自研评估框架 |
| 自动化决策流 | 从特征→模型→规则→策略的自动化 Pipeline | Airflow / Temporal |

**核心功能**：
- 可视化特征工程工作台
- 模型训练与版本管理
- 规则策略可视化编辑与生成
- 策略仿真与效果预估
- 策略生命周期管理

### 2.4 执行层 (Execution Layer)
> **定位**：将决策策略安全、可靠地下发到业务系统

| 能力模块 | 描述 | 技术方向 |
|---------|------|---------|
| 策略分发引擎 | 将规则策略转换为业务系统可执行的格式 | API Gateway |
| 风控系统适配器 | 对接风控引擎，实时下发风控策略 | 适配器模式 |
| 营销系统适配器 | 对接营销平台，下发营销策略 | 适配器模式 |
| 执行监控 | 策略执行日志、命中率监控、异常告警 | Prometheus + Grafana |
| 策略回滚 | 支持策略快速回滚与灰度发布 | 灰度策略框架 |

**核心功能**：
- 多目标系统适配器管理
- 策略下发状态追踪
- 实时执行监控与告警
- 灰度发布与回滚

### 2.5 应用层 (Application Layer)
> **定位**：面向最终用户的产品应用

| 产品 | 描述 | 目标用户 |
|------|------|---------|
| **AIbi** | AI 驱动的智能商业分析产品，支持自然语言查询与分析 | 业务分析师、管理者 |
| **数据可视化产品** | 用户自助数据可视化，支持拖拽式图表与仪表盘 | 全员 |
| **本体探索器** | 知识图谱可视化浏览与查询 | 数据工程师 |
| **策略工作台** | 策略创建、编辑、测试、发布的一站式工作台 | 策略分析师 |

**核心功能**：
- AIbi：自然语言→SQL/图谱查询→可视化
- 数据可视化：拖拽式仪表盘、实时数据刷新
- 本体探索器：图谱交互式探索
- 策略工作台：低代码策略编辑

## 六、核心设计原则

1. **层次解耦**：每层独立部署、独立扩展，通过 API 通信
2. **AI 原生**：LLM 深度嵌入各层，从数据感知到策略生成全流程 AI 增强
3. **本体驱动**：以知识图谱为数据中枢，连接感知层输入与决策层输出
4. **可观测性**：全链路日志、监控、追踪
5. **安全合规**：数据脱敏、权限管控、审计日志
