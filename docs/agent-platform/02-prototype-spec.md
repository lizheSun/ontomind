# Agent 资源管理平台交互原型说明

> 状态：待原型确认  
> 原型路由：`/prototype/agent-platform`  
> 数据说明：全部使用前端模拟数据，不连接 SSH、OpenCode、Agent 或发布接口。

## 1. 原型目标

本原型用于确认四个核心工作区的信息架构和关键交互：

1. 对话工作台是否适合作为默认首页。
2. 资源发现和远程节点注册流程是否容易理解。
3. Agent Studio 的显式设计方法论是否足够清晰。
4. 发布、下线和回滚是否形成完整生命周期。

原型不用于验证视觉细节、接口性能或技术可行性，这些内容在技术方案和实施阶段处理。

## 2. 原型入口

登录项目后访问：

```text
/prototype/agent-platform
```

页面顶部可切换：

- 对话工作台
- 资源控制台
- Agent Studio
- 发布中心

原型没有改动现有默认首页和正式资源导航，确认后再决定正式替换方式。

## 3. 对话工作台

### 布局

- 左侧：Agent、状态、版本和最近会话。
- 中间：对话消息和输入区。
- 右侧：当前 Run 的结构化执行轨迹。

### 可操作内容

- 切换不同 Agent。
- 发送模拟消息。
- 查看模拟工具调用及结果。
- 批准或拒绝待审批工具。
- 清空会话。
- 从对话头部进入 Agent Studio。

### 重点验证

- 三栏结构是否清晰。
- Agent 与会话放在同一栏是否合适。
- 执行轨迹默认常驻还是应默认收起。
- 思考摘要、步骤、工具和 Eval 是否应该在右侧统一展示。

## 4. 资源控制台

### 布局

- 顶部五类资源统计。
- 中间节点表格。
- 节点、Runtime、Agent、Skill、MCP 分层切换。

### 节点注册向导

点击“注册计算节点”，依次体验：

1. 本机或远程 Linux。
2. 凭据写入或引用。
3. SSH 主机指纹确认。
4. 权限检查。
5. OpenCode 发现预览。

### 发现结果

点击“扫描”或“全量发现”，查看：

- 新发现。
- 已关联。
- 有差异。
- 缺失。
- 对应的导入、查看差异和人工处理动作。

### 重点验证

- “Agent Runtime”术语是否明确。
- 五步节点向导是否过长。
- 主机指纹确认是否需要独立步骤。
- 发现结果是否应该在抽屉、独立页面或节点详情中处理。
- 冲突动作是否足够清楚。

## 5. Agent Studio

### 布局

- 左侧：十步 Agent 设计方法。
- 中间：当前步骤表单。
- 右侧：方法解释、配置完整度和风险提示。

### 可操作内容

- 快速模式与专家模式切换。
- 十个步骤自由跳转。
- 体验目标、协作和 Eval 三个重点步骤。
- 顺序执行上一步和下一步。
- 进入发布检查。

### 十步方法

1. 目标
2. 角色
3. 模型
4. 能力
5. 协作
6. Loop
7. Hook
8. Eval
9. 护栏
10. 发布

### 重点验证

- 十步是否过多。
- 快速模式应合并哪些步骤。
- 协作模式选择是否比直接画 DAG 更容易理解。
- Eval 配置是否需要与调试用例放在同一个步骤。
- 风险和准备度是否应该常驻右侧。

## 6. 发布中心

### 布局

- 待发布版本和配置 diff。
- 发布 Eval 门禁。
- 目标环境与节点。
- 发布操作和结果。

### 可操作内容

- 选择测试或生产目标。
- 查看依赖和 Eval 状态。
- 模拟发布进度。
- 发布后进入下线或回滚入口。

### 重点验证

- Agent Version 与 Deployment 是否容易区分。
- 测试和生产发布是否使用同一页面。
- 下线和回滚应放在发布中心还是 Agent 详情。
- 生产发布审批是否应该独立成“待办”。

## 7. 八条核心旅程覆盖

### 已完整表达

1. 注册本机。
2. 注册远程 Linux。
3. 发现 OpenCode。
4. 处理配置冲突。
5. 创建和配置 Agent。
6. 对话调试和工具审批。
7. Eval 后发布。
8. 下线或回滚入口。

### 仅表达入口、未展开完整页面

- 长任务暂停和恢复。
- 审计检索。
- 版本历史详情。
- 多目标发布的逐节点失败处理。
- 从 checkpoint 恢复。

这些内容在原型确认后进入技术方案和后续详细原型。

## 8. 复用与变化

### 复用现有能力

- `AgentChatPanel` 的消息和工具审批原语。
- 现有 Ant Design 暗色主题。
- 资源页面的统计卡、表格和详情风格。
- 已有 Agent Looper 的模板、Loop 和 Eval 概念。

### 原型提出的主要变化

- 默认工作中心从静态仪表盘改为对话。
- “智能体容器”在产品界面改称“Agent Runtime”。
- Agent 配置改为十步方法，而不是直接编辑 JSON。
- 发现只生成预览，导入和冲突处理必须人工确认。
- 发布对象明确为 Agent Version 到目标 Runtime 的 Deployment。

## 9. 本次确认清单

确认原型时，请重点判断：

- 对话工作台是否作为未来默认首页。
- 四个一级工作区是否足够，是否还需要单独的运行中心和审计中心原型。
- 对话三栏布局是否保留。
- 节点注册五步向导是否保留。
- 发现结果是否使用右侧抽屉。
- Agent Studio 十步是否保留。
- 快速模式/专家模式是否保留。
- 协作模式优先于任意 DAG 编辑器是否可接受。
- 发布中心的版本、门禁、目标和操作四区布局是否保留。

原型确认后，下一阶段将编写数据模型、SSH Connector、安全、事件协议、迁移和测试技术方案，不会立即实现真实远程执行。
