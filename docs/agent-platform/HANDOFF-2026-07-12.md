## 0. 一句话总结

今天落地了 **Agent 资源管理 + Studio + 对话工作台**，并把对话从「整段阻塞返回」改成 **OpenCode 流式执行 + SSE 实时推送**（thinking / tools / steps / message.delta）。

---

## 1. 今天干了什么（按产品能力）

| 能力 | 说明 | 入口 |
|------|------|------|
| 资源控制台 | 本机自动 `register-local` + OpenCode inventory；三栏：节点 / OpenCode+资源 / 详情；平台 Agent 编辑/发布/去对话 | `/agent-platform/resources` |
| Agent Studio | 快速模式配置 Agent；保存草稿/发布；自动绑本机 OpenCode runtime | `/agent-platform/agents/new/studio` |
| 对话工作台 | 选 Agent 聊天；列表内不做编辑发布（改去资源管理） | `/workspace` |
| 运行记录 | Run 列表 + SSE 时间线 | `/agent-platform/runs` |
| 真实 OpenCode | `opencode run --format json`，按行解析 JSONL | 后端 `OpenCodeChatService` |
| 流式 SSE | 发消息立刻返回 `run_id`；后台流式写事件；SSE 长连接推送 | `POST .../sessions/{id}/messages` + `GET .../runs/{id}/events` |

### 产品层级（已确认）

```
计算节点 → OpenCode 容器 → Agent / Skill / MCP
```

- Agent **真源**：`agents` 表（平台侧）
- OpenCode：DB 做控制面；冲突人工确认
- **编辑/发布在资源管理**；对话工作台只聊天

## 2. 流式架构（最新关键改动）

```
浏览器 POST /sessions/{id}/messages
  → 立刻返回 { run_id, queued: true }
  → BackgroundTasks 执行 RunService.execute()
       → OpenCodeChatService.stream_prompt() 按行读 JSONL
       → map_opencode_event() → agent_run_events（每条 commit）
浏览器 GET /runs/{id}/events (SSE 长连接)
  → 轮询新事件直到 run.completed / failed / cancelled
前端 useAgentStream + timelineReducer
  → 聊天气泡 message.delta + 右侧 thinking/tool/step
```

测试仍可用 `metadata.force_stub=true` **同步**跑 stub（不调 OpenCode）。
