# 指标命名规范

## 一、指标 ID

- 格式：`M` + 4 位数字，例如 `M0001`
- 一级指标：M0001–M0099
- 二级指标：M0100–M0499
- 三级指标：M0500–M9999
- **不允许重用**：废止的 ID 永久占用

## 二、中文名 (name_zh)

- 长度：2–20 个字
- 必须**动宾结构**或**名词短语**
- 禁用形容词修饰（如 "很高" "重要"）
- ✅ 正确：`日活跃用户数`、`授信通过率`、`M1 逾期率`
- ❌ 错误：`最重要的日活`、`很高的通过率`

## 三、英文名 (name_en)

- 全小写 + snake_case
- 长度 ≤ 32
- 单词精简（用业界通用缩写）
- ✅ 正确：`dau`, `approval_rate`, `m1_overdue_rate`, `ltv`
- ❌ 错误：`Daily_Active_User_Count`, `TheApprovalRate`

## 五、别名规则 (aliases)

- 每个指标至少配 **2 个别名**
- 别名类型：
  1. 业务黑话/口语（如 `日活`）
  2. 英文缩写（如 `DAU`）
  3. 历史遗留字段名（如 `active_users_daily`）
  4. 跨部门叫法（如 财务部叫 `活跃客户数`）

## 六、口径命名（filters / dedup_key / time_grain）

- `filters`：过滤条件写成人类可读的清单
  - ✅ `is_test_user = 0`
  - ❌ `p1`（含义不清）
- `dedup_key`：去重维度显式化
  - ✅ `user_id`（账号维度）
  - ✅ `device_id`（设备维度）
- `time_grain`：`day` / `week` / `month` / `quarter` / `year` / `rolling_30d` / `rolling_90d`

## 七、命名反模式

| ❌ 反模式 | ✅ 推荐 |
|---|---|
| `total_amount` | `disburse_amount`（说清主体） |
| `count` | `apply_count` / `loan_count` |
| `rate` | `approval_rate` / `overdue_rate_m1` |
| `today` | `daily`（time_grain） |
| `很多用户` | `active_users` |

10. `overdue_rate_m3` — M3 逾期率
11. `interest_income` — 利息收入
12. `fee_income` — 手续费收入
13. `total_income` — 总收入
14. `net_interest_margin` — 净息差
15. `cost_of_fund_rate` — 资金成本率
16. `cpa` — 单个获客成本
